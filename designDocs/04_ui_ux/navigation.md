# Navigation

## 1. Introduction
La navigation décrit comment l’utilisateur passe d’un écran à un autre et interagit avec l’application.

## 2. Menu principal
- **Accueil** : point de départ pour rechercher un trajet  
- **Favoris** : accès rapide aux trajets sauvegardés  
- **Historique** : consultation des trajets précédents  
- **Paramètres** : gestion du profil et des préférences  

## 3. Flux utilisateur
1. L’utilisateur saisit le **point de départ et d’arrivée** sur l’écran d’accueil.  
2. Il appuie sur **Rechercher** → écran des résultats.  
3. Il peut sélectionner un trajet pour voir les **détails** → écran de détails du trajet.  
4. Il peut ajouter le trajet aux **favoris** ou revenir à l’écran des résultats.  
5. Depuis n’importe quel écran, l’utilisateur peut accéder au **menu principal** pour naviguer vers **favoris, historique ou paramètres**.

## 4. Navigation mobile
- **Barre de navigation inférieure** pour les sections principales : Accueil, Favoris, Historique, Paramètres  
- **Boutons retour** intégrés dans chaque écran pour faciliter le parcours utilisateur

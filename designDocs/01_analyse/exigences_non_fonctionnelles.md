# Exigences non fonctionnelles

## 1. Performance
- L'application doit répondre rapidement aux requêtes (moins de 2 secondes pour afficher les résultats).  

## 2. Sécurité
- Protection des données personnelles et de localisation.  
- Communication sécurisée avec les API (HTTPS).  

## 3. Disponibilité
- L'application doit être accessible **24h/24 et 7j/7**.  

## 4. Compatibilité
- Fonctionnement sur **Android** et **iOS**.  
- Adaptation aux différentes tailles d’écran de smartphone.  

## 5. Scalabilité
- Capacité à gérer un **grand nombre d’utilisateurs simultanés**.  

## 6. Maintenabilité
- Code et architecture **faciles à mettre à jour et à faire évoluer**.
